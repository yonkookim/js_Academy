position 요소, 노드 개체의 위치를 정의한다.(static, relative, absolute, fixed, sticky)

position: static(default)

모든 요소, 노드 개체는 위치를 static을 기본으로 정의한다.

  • 요소의 위치는 순차적이며 왼쪽->오른쪽 순으로 배치
  • 자식요소의 위치는 기본적으로 부모요소의 위치를 기준으로 배치된다.
  • top, bottom, left, right를 지정 할 수 있다.
  • 다음 수행에서 애니메이션 효과가 조금 이상하게 먹는 것을 확인 할 수 있는데.. 이유는?
  • transition-property을 설정할 때 해당 노드에 top. left 속성의 설정이 들어가 있어야 효과가 먹힌다.
  • 아.. 그래서 두번 이동하고 원점 이동할 때는 속성이 들어간 뒤이니 효과가 먹힌 것

position: relative(상대적 위치)

relative


  • position: relative는 static을 기준으로 노드를 이동시킬 수 있게 해준다.

position: absolute(절대적 위치)

  • '부모 노드'의 위치 설정이 static일때는
  • position: absolute인 자식노드 위치의 기준점은 root(최상위 부모) 노드인 body

  • '부모 노드'중에 위치가 static이 아닌 노드가 있을땐?
  • position: absolute인 자식노드 위치의 기준점은 해당하는 그 '부모 노드'가 된다.
absolute

position: absolute(부모의 위치가 static이 X)

absolute


position: fixed

  • 부모 요소와 관계없이 viewport(브라우저화면)을 기준으로 개체 위치를 고정시킨다.
  • fixed 역시 absolute 처럼 개체의 넓이와 높이가 content에 맞게 변경된다.
  • fixed 역시 absolute 처럼 순차적 흐름에서 벗어나 부유객체가 된다.
