JavaScript js 소개

자바스크립트는 웹 브라우저 상에서 UI를 동적으로 보여주기 위하여 개발 되었으나 배우기 쉽고 개발 속도가 빨라서 서버나(Node.js) 모바일앱 (react-native, NativeScript)등 프로그래밍의 여러 분야에서 사용되고 있다. 또한 매해 새 버전을 출시해 오류를 수정하거나 객체지향적 특성을 발전시키고 있다.

브라우저 언어인 자바스크립트의 특징

  • 자바스크립트는 객체기반의 스크립트 언어로 브라우저에서 사용된다. (* 스크립트 언어: 응용프로그램(브라우저)을 동작시키는 언어의 통칭)
  • 자바스크립트는 인터프린터 언어로 컴파일 되지 않고 바로 해석된다. (* 컴파일: 작성한 코드를 기계어로 해석하는 행위, 오류가 있는 코드는 컴파일 되지 않는다.) 때문에 동적으로 빠르게 개발할 수 있고 오류가 있는 페이지도 배포할 수 있다. 실행속도는 느리지만 인지하기 힘들차이.
  • 자바스크립트는 함수형 프로그래밍 언어였으나 버전 es6이후로 class와 변수를 추가해 객체지향 언어의 특징을 가진다.
  • 자바스크립트는 주로 다루는 객체가 window.document로 DOMTree로 구성된 document의 특정 Element를 찾는 것에서부터 시작된다. 예)document로.getElementById()
  • 자바스크립트는 html문서가 모두 브라우저에 출력되고 마지막에 실행된다.

자바스크립트가 브라우저에서 할 수 있는 일

  • html문서의 내용, 속성을 추가 수정 삭제 할 수 있다.
  • html 문서의 스타일을 변경할 수 있다.
  • html요소에서 발생하는 외부기기의 입력을 이벤트로 받아 ui를 동적으로 바꾼다.
  • 브라우저의 (BOM) 정보를 알 수 있거나 입력 및 수정을 할 수 있다.

html에서 자바스크립트 작성법

  • 코드는 script 태그 내부에서 작성한다.
  • script src="" 확장자가 js인 파일을 외부에서 참조할 수 있다.
  • 코드를 실행 및 선언 후 꼭! 세미콜론 ; 을 작성해야 한다.
  • // 한줄 주석
  • /**/ 여러줄 주석
  • "use strict" : 엄격모드 (es6문법 사용시 자동으로 설정됨, class let...등) js파일 상단에 꼭 작성해야 한다.(뭔소리?) 이유는 과거 문법이 오류를 많이 만들기 때문 예) var num=10; (o) num=20;(o,x) use strict모드에서는 이런것을 인정하지 않는다.
